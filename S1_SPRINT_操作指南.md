## 🎯 执行步骤(逐集进行)

### 通用流程(对EP2-EP10每一集重复)

#### Step 1: 切换到目标集
- 在左侧剧集列表中,点击 **EP2**
- 等待右侧面板加载完成

#### Step 2: 确认当前状态
- 查看右侧顶部状态标签
- 状态应该是 **DRAFT** 或 **DEGRADED**
- 如果是 **COMPLETED**,说明已经处理过了

#### Step 3: 应用"加重代价"指令
- 在右侧面板找到 **"微调指令"** 区域
- 点击 **"加重代价"** 按钮(标签:TrendingUp 图标,绿色)
- 等待提示: **"微调指令已应用,剧本已重新生成"**

#### Step 4: 阅读并验收

##### 📝 验收清单

阅读重新生成后的内容,检查以下3个问题:

1. **✅ 是否有明确的损失/风险/付出?**
   - 失去了什么?
   - 被伤害了什么?
   - 背负了什么代价?

2. **✅ 主角或关键角色的状态是否变化?**
   - 身份地位变化
   - 关系格局变化
   - 资源/能力变化

3. **✅ 这一集结尾是否"更难了"而不是"更顺了"?**
   - 局势是否更加困难
   - 压力是否增加
   - 是否有新威胁出现

##### 🎯 核心验收标准

**能否用一句话说清:"这一集他失去了什么/背上了什么?"**

如果能 → **通过验收**

如果不能 → **重新应用"加重代价"指令** 或 **手动编辑内容**

#### Step 5: 记录结果(可选)

在纸上或记录器中:
- EP2: ✓ 通过 / ✗ 需要优化
- EP3: ✓ 通过 / ✗ 需要优化
- ...

## ✅ 最终验收

完成所有9集后,检查Metrics报告:

### 查看方式
1. 检查 `reports/` 目录
2. 查找最新的 `m16_metrics_*.json` 文件
3. 打开文件查看 `aggregates.health` 部分

### 验收标准

```
revealIsConcrete ≥ 60%    ✓ 通过
revealHasConsequence ≥ 70%  ✓ 通过
DEGRADED ≤ 3               ✓ 通过
```

### 判断结果

如果3项全部达标 → **✓✓✓ S1 Sprint 成功!**

如果有部分未达标 → **✗✗✗ 部分达标,需要进一步优化**

## 🔄 故障排查

### 问题1: 点击按钮没反应
- 检查浏览器控制台是否有错误
- 刷新页面重试
- 检查网络连接

### 问题2: 重新生成后状态还是COMPLETED
- 这是正常的,COMPLETED也可以重新生成
- 继续验收内容质量

### 问题3: 内容没有变化
- 检查是否有错误提示
- 尝试刷新页面
- 查看浏览器控制台日志
